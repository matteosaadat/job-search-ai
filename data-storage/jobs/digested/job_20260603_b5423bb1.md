## Summary

Intertech is staffing a remote contract front-end developer role for an undisclosed enterprise client undergoing a large-scale UI modernization. The work involves replacing a legacy UI with a modern React/TypeScript stack across 300+ screens, working alongside backend engineers as the system migrates to a service-based architecture.

The stack is a strong match: React, TypeScript, Tailwind CSS, and TanStack are all modern, well-aligned tools. The scope is substantial — 300+ screens of medium complexity — suggesting a multi-month engagement with real impact. Agile process, REST/GraphQL integration, and accessibility compliance round out the work.

Rate is $70–$100/hr depending on W2, 1099, or C2C engagement. The floor is below target but the ceiling ($100/hr C2C) is within range. The client is undisclosed, which limits due diligence on the actual end-company.

## Key Requirements

- React + TypeScript (strong, not just exposure)
- Tailwind CSS
- TanStack Query and/or Router
- REST API integration; GraphQL a plus
- 300+ screen modernization capacity — someone who can work systematically at scale
- WCAG 2.0/2.1 accessibility awareness
- Agile/sprint workflow

## Responsibilities

- Build and modernize React/TS UI components across 300+ screens
- Replace legacy UI framework with modern stack
- Integrate with backend REST and GraphQL services
- Align UI architecture with microservices patterns
- Participate in Agile ceremonies
- Support accessibility compliance

## Red Flags

- Rate floor $70/hr W2 is below $85/hr minimum — only viable at $90–100/hr C2C or 1099
- Client undisclosed — can't vet the actual company or project stability
- Intertech is a staffing intermediary — they have margin on the rate
- No duration stated — modernization projects can run long or get de-funded mid-stream
- Government sector exposure preferred — may indicate the end client is a gov agency (slower, more process-heavy)
